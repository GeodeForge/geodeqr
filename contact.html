Why this is a mailto and not a form

Because a form needs a server, and this site doesn't have one. That's not laziness — it's the whole point. The page loads its own HTML, CSS and scripts and then goes quiet: once it has loaded, making a code fires no requests at all, and nothing you type ever leaves your machine. A contact form would break that, so contact lives here instead of on the tool.

If you'd rather not use email, the source is on GitHub — open an issue there.
